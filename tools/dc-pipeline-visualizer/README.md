# dc-pipeline-visualizer

`dc-pipeline-visualizer` tails `metrics.jsonl` produced by `xtask local`, tracks queue/task metric history in-memory, and streams live updates to a browser UI over WebSocket.

## What it does

The server consumes the existing JSONL metric rows from `xtask local` (including `metric`, `type`, `process`, `ts`, `run`, and `workloads`) and emits a stable WebSocket message contract:

- `bootstrap`: graph config + full in-memory snapshot for newly connected clients
- `metrics`: incremental metric updates
- `health`: backpressure and stream health messages
- `state`: server state transitions

The browser UI (React + Vite + Tailwind) renders a declarative node/edge graph, overlays queue/task metrics, supports run/workload/process filtering, and highlights bottlenecks.

## Graph config

Use JSON to define connectivity and metric selectors:

- `nodes[]`: `id`, `label`, optional `kind`, optional `x`/`y`, `selectors[]`
- `edges[]`: `id`, `from`, `to`, optional `label`, `selectors[]`
- selector fields: `metric_exact`, `metric_prefix`, `metric_type`, `process`, `run`, `workload`, `variant`

A starter file is provided at:

- `/home/runner/work/s2n-quic/s2n-quic/tools/dc-pipeline-visualizer/graph.example.json`

## Local workflow

1. Run workload and produce metrics JSONL:

   ```bash
   cd /home/runner/work/s2n-quic/s2n-quic
   cargo run -p xtask -- local --workloads <workload>
   ```

2. Build the UI:

   ```bash
   cd /home/runner/work/s2n-quic/s2n-quic/tools/dc-pipeline-visualizer/ui
   npm install
   npm run build
   ```

3. Start the visualizer server:

   ```bash
   cd /home/runner/work/s2n-quic/s2n-quic
   cargo run -p dc-pipeline-visualizer -- \
     --metrics-file /home/runner/work/s2n-quic/s2n-quic/logs/<workload>/<run>/metrics.jsonl \
     --graph-config /home/runner/work/s2n-quic/s2n-quic/tools/dc-pipeline-visualizer/graph.example.json
   ```

4. Open `http://127.0.0.1:3000`.

## CLI

- `--metrics-file <PATH>`: JSONL stream generated by `xtask local`
- `--graph-config <PATH>`: graph topology and selector file
- `--bind <ADDR>`: server bind address (default `127.0.0.1:3000`)
- `--history-seconds <N>`: in-memory history window for sparklines (default `90`)
- `--ui-dist <PATH>`: built UI assets directory (default `ui/dist`)
